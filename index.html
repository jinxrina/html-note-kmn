Basic Forms & Input
Build interactive forms with basic input types, proper structure, and form submission.

üìö Key Learning Points:

  • <form> element defines the form boundary and submission behavior
  • action attribute specifies where form data is sent
  • method attribute defines HTTP method (GET/POST)
  • <label> elements improve accessibility and usability
  • name attributes are essential for server-side processing
<form action="/submit" method="POST" id="contact-form">
    <!-- Text Input -->
    <label for="name">Full Name:</label>
    <input type="text" id="name" name="fullname" required placeholder="Enter your full name">
    
    <!-- Email Input -->
    <label for="email">Email Address:</label>
    <input type="email" id="email" name="email" required>
    
    <!-- Password Input -->
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" minlength="8">
    
    <!-- Submit Button -->
    <button type="submit">Submit Form</button>
    <button type="reset">Reset</button>
</form>

HTML5 Input Types
Explore modern HTML5 input types for better user experience and built-in validation.

üìö Key Learning Points:

  • HTML5 input types provide automatic validation and mobile keyboards
  • date, time, datetime-local for temporal data entry
  • number, range for numeric inputs with min/max constraints
  • color picker, file upload, and search inputs
  • tel and url inputs trigger appropriate mobile keyboards
<form>
    <!-- Date and Time Inputs -->
    <label for="birthday">Birthday:</label>
    <input type="date" id="birthday" name="birthday" min="1900-01-01" max="2024-12-31">

    <label for="appointment">Appointment:</label>
    <input type="datetime-local" id="appointment" name="appointment">

    <!-- Numeric Inputs -->
    <label for="age">Age:</label>
    <input type="number" id="age" name="age" min="13" max="120" step="1">

    <label for="volume">Volume:</label>
    <input type="range" id="volume" name="volume" min="0" max="100" value="50">

    <!-- Color and File Inputs -->
    <label for="favcolor">Favorite Color:</label>
    <input type="color" id="favcolor" name="favcolor" value="#3b82f6">

    <label for="avatar">Profile Picture:</label>
    <input type="file" id="avatar" name="avatar" accept="image/*">

    <!-- Contact Inputs -->
    <label for="phone">Phone:</label>
    <input type="tel" id="phone" name="phone" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}" placeholder="123-456-7890">

    <label for="website">Website:</label>
    <input type="url" id="website" name="website" placeholder="https://example.com">

    <!-- Search Input -->

Form Validation & Attributes
Implement client-side validation with HTML5 attributes and provide user feedback.

üìö Key Learning Points:

  • required attribute makes fields mandatory
  • pattern attribute uses regex for custom validation
  • min, max, step control numeric input ranges
  • maxlength, minlength limit text input length
  • Custom validation messages improve user experience
<form novalidate>
    <!-- Required Field with Custom Message -->
    <label for="username">Username (required):</label>
    <input type="text" 
           id="username" 
           name="username" 
           required
           minlength="3"
           maxlength="20"
           pattern="[a-zA-Z0-9_]+"
           title="Username must be 3-20 characters, letters, numbers, and underscore only">

    <!-- Email with Custom Pattern -->
    <label for="work-email">Work Email:</label>
    <input type="email" 
           id="work-email" 
           name="workemail"
           pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$"
           title="Please enter a valid email address">

    <!-- Password with Strength Requirements -->
    <label for="strong-password">Strong Password:</label>
    <input type="password" 
           id="strong-password" 
           name="password"
           required
           minlength="8"
           pattern="(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}"
           title="Password must be 8+ chars with uppercase, lowercase, number, and special character">

    <!-- Numeric Range Validation -->
    <label for="price">Price ($):</label>
    <input type="number" 
           id="price" 
           name="price"
           min="0.01"
           max="999999.99"
           step="0.01"
           required
           placeholder="0.00">

    <!-- File Upload with Restrictions -->
    <label for="document">Document (PDF only):</label>
    <input type="file" 
           id="document" 
           name="document"
           accept=".pdf"
           required>

    <!-- Agreement Checkbox -->
    <label>
        <input type="checkbox" name="agree" required>
        I agree to the terms and conditions
    </label>

    <button type="submit">Submit with Validation</button>
</form>

Form Accessibility & UX
Create accessible forms with proper labeling, error handling, and user-friendly design.

üìö Key Learning Points:

  • Every form control must have an associated label
  • Use aria-describedby for additional help text
  • Provide clear error messages and success feedback
  • Group related fields with fieldset and legend
  • Ensure keyboard navigation works properly
<form role="form" aria-labelledby="form-title">
    <h2 id="form-title">Contact Information</h2>
    
    <!-- Accessible Input with Help Text -->
    <div class="form-group">
        <label for="email-acc">Email Address *</label>
        <input type="email" 
               id="email-acc" 
               name="email"
               required
               aria-describedby="email-help email-error"
               aria-invalid="false">
        <div id="email-help" class="help-text">
            We'll use this to send you important updates
        </div>
        <div id="email-error" class="error-message" role="alert" aria-live="polite">
            <!-- Error message will appear here -->
        </div>
    </div>

    <!-- Fieldset for Related Options -->
    <fieldset>
        <legend>Preferred Contact Method *</legend>
        <div role="radiogroup" aria-required="true">
            <label>
                <input type="radio" name="contact-method" value="email" required>
                Email
            </label>
            <label>
                <input type="radio" name="contact-method" value="phone" required>
                Phone
            </label>
            <label>
                <input type="radio" name="contact-method" value="text" required>
                Text Message
            </label>
        </div>
    </fieldset>

    <!-- Multi-line Input with Character Count -->
    <div class="form-group">
        <label for="message-acc">Your Message</label>
        <textarea id="message-acc" 
                  name="message"
                  rows="4"
                  maxlength="500"
                  aria-describedby="message-count"
                  placeholder="Tell us how we can help you..."></textarea>
        <div id="message-count" class="char-count">
            0/500 characters
        </div>
    </div>

    <!-- Submit Button with Loading State -->
    <button type="submit" 
            aria-describedby="submit-help">
        Send Message
    </button>
    <div id="submit-help" class="help-text">
        Your message will be sent securely
    </div>

    <!-- Success Message Area -->
    <div id="success-message" class="success-message" role="status" aria-live="polite">
        <!-- Success message will appear here -->
    </div>
</form>

Web Components
Create reusable custom elements using modern web component technologies and shadow DOM. 
<!-- Custom Element Definition -->
<script>
class MyButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({mode: 'open'});
    this.shadowRoot.innerHTML = `
      <style>
        button { padding: 10px; background: blue; color: white; }
      </style>
      <button><slot></slot></button>
    `;
  }
}
customElements.define('my-button', MyButton);
</script>

<!-- Using Custom Element -->
<my-button>Click me!</my-button>

<!-- Template Element -->
<template id="card-template">
    <div class="card">
        <slot name="title"></slot>
        <slot></slot>
    </div>
